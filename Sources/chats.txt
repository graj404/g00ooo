        #Option 1 — Steering Wheel Rotation Sensor (SAS)#
        A Steering Angle Sensor sits on the steering column and measures how many degrees the wheel has rotated.
        How it works:

        Measures steering wheel rotation (typically ±720° or more — multiple full turns)
        You then divide by the steering ratio (e.g. 15:1) to get the actual front wheel angle
        front_wheel_angle = steering_wheel_angle / steering_ratio

        Sensor types used:

        Optical encoder — a disc with slots, counts pulses as it rotates
        Hall effect sensor — detects magnetic field changes as a magnet rotates past it
        Resolver — absolute angle measurement, very precise, used in modern cars

        Pros: Easy to mount, doesn't move with the suspension, already standard in all modern cars (required for ESP/ESC systems)
        Cons: You must know/calibrate the steering ratio, and you need to find the center (zero) point on startup

        Method 2 — GPS Doppler Velocity
            A GPS module doesn't just give position — it gives velocity directly via the Doppler shift of satellite signals.
            How it works:

            The GPS chip measures the rate of change of distance to each satellite using signal frequency shift
            This gives a velocity vector (speed + direction) independently of position
            Update rate: typically 1–10 Hz (up to 25 Hz on high-end modules like u-blox F9)

            Pros: Absolute ground truth speed, no wheel slip error, free if you already have GPS
            Cons: Low update rate (10 Hz max typically), fails in tunnels/urban canyons, ~0.1 m/s accuracy

        Method 3 — IMU Double Integration (Accelerometer)
            An IMU (Inertial Measurement Unit) measures acceleration, which you integrate over time to get velocity.
            How it works:
            v(t) = v(t-1) + a(t) × dt

            Accelerometer gives a(t) at very high rates (100–1000 Hz)
            Integrate once → velocity
            Integrate twice → position

            Pros: Extremely high update rate (1000 Hz possible), works anywhere, no external dependency
            Cons: Drift — small errors in acceleration accumulate over time, so velocity drifts badly after ~30 seconds without correction

        Method 4 — Optical Flow / Visual Odometry
            A downward-facing camera or laser looks at the ground and measures how fast features move.
            How it works:

            Like how a computer mouse works — tracks texture movement on the ground
            Used in drones, AGVs, and some autonomous vehicles
            More advanced version: Visual Odometry using a forward-facing camera tracking scene features

            Pros: True ground speed (immune to wheel slip), no GPS needed
            Cons: Fails on featureless surfaces (wet road, snow), needs clean sensor

        Method 5 — Radar / Lidar Doppler
            High-end vehicles use radar that directly measures ground-relative velocity via Doppler effect.
            How it works:

            Radar emits a signal, the reflection frequency shift reveals relative velocity to the ground
            Used in adaptive cruise control radars, but also measurable against static objects

            Pros: Most accurate ground speed, immune to all wheel conditions
            Cons: Expensive, overkill for most projects

            What Real Systems Actually Do — Sensor Fusion
            No single sensor is used alone. The standard approach fuses multiple sources:
            Wheel RPM   ──┐
            GPS Doppler ──┤──► Kalman Filter ──► Best velocity estimate
            IMU accel   ──┘

            Wheel RPM gives fast, high-frequency updates
            GPS corrects drift every second
            IMU fills gaps when GPS drops out
            A Kalman Filter weighs each source by its current reliability


        Practical Recommendation for Your Project
            For your car path tracker, the simplest effective stack is:
            Tier 1 (easiest): Read wheel speed from OBD-II / CAN bus — speed is already computed and broadcast at 100 Hz on every modern car. Just plug in an ELM327 or MCP2515 module.
            Tier 2 (more accurate): Add a GPS module (u-blox NEO-M8N, ~$15) and fuse GPS Doppler velocity with wheel speed to cancel slip errors.
            Tier 3 (research grade): Add an IMU (MPU-6050 or ICM-42688) and run a simple complementary filter or Kalman filter across all three sources.
